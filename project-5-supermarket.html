02 The Challenge — Data Integrity

During the initial data audit, a critical problem was discovered: many products were severely misclassified. For example, 'Apple' was labeled as 'Beverages', and 'Shampoo' appeared under 'Food'. Over 70% error rate was found in specific product samples.

Without fixing this, any revenue analysis would be completely misleading. A Python cleaning script was written using Pandas to re-map every product to its correct category before any analysis began.

Why Data Cleaning Came First

Running analysis on dirty data produces confident-looking but completely wrong conclusions. Fixing the data integrity issue first meant the final insights were actually trustworthy — and directly contradicted what a naive analysis would have shown.

04 Python Implementation

The actual Python code used to clean the dataset and perform Exploratory Data Analysis (EDA). The data correction block was the critical first step before any analysis:

Data Cleaning + EDA — sales.csv Python 
import pandas as pd
import matplotlib.pyplot as plt

# ─── 1. Load Data ───────────────────────────────────────────────────
sales = pd.read_csv('sales.csv')

print(f"Dataset shape: {sales.shape}")
print(sales['product_category'].value_counts())

# ─── 2. Data Cleaning & Correction (CRITICAL STEP) ──────────────────
# Audit revealed mismatches between Product Name and Category.
# e.g. 'Apple' labeled as 'Beverages' — clearly wrong.
# Creating corrected dataset 'new_sales'.

new_sales = sales.copy()

# Re-map each misclassified product to its correct category
new_sales.loc[new_sales['product_name'] == 'Apple',        'product_category'] = 'Fruits'
new_sales.loc[new_sales['product_name'] == 'Orange Juice', 'product_category'] = 'Beverages'
new_sales.loc[new_sales['product_name'] == 'Notebook',     'product_category'] = 'Stationery'
new_sales.loc[new_sales['product_name'] == 'Detergent',    'product_category'] = 'Household'
new_sales.loc[new_sales['product_name'] == 'Shampoo',      'product_category'] = 'Personal Care'
# ... (full mapping in GitHub repo)

# ─── 3. Analysis on Cleaned Data ────────────────────────────────────

# Revenue by Product Category (after cleaning)
category_sales = (
    new_sales
    .groupby('product_category')['total_price']
    .sum()
    .sort_values(ascending=False)
)
print("\nTop Categories by Revenue (CLEANED):\n", category_sales)

# Revenue distribution by City
sales.groupby('city')['total_price'].sum().plot(
    kind='pie', autopct='%1.1f%%', figsize=(8, 8)
)
plt.title("Sales Distribution by City")
plt.ylabel('')
plt.tight_layout()
plt.show()

# ─── 4. Customer Segmentation ───────────────────────────────────────
# Identify most valuable customer segments
valuable_customers = (
    sales
    .groupby(['gender', 'customer_type'])['total_price']
    .mean()
    .sort_values(ascending=False)
)
print("\nAvg Spend by Segment:\n", valuable_customers)
# Result: Female Members spend the most on average.

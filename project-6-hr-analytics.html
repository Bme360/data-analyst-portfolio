02 Project Overview

This project simulates a real-world recruitment analytics pipeline. The dataset was generated synthetically using Python to replicate realistic hiring dynamics — candidates with more experience and higher LinkedIn scores have a statistically higher probability of being hired.

Dataset

SQL Server

Power BI

Key Goal

Answer: What drives hiring success? Does experience, education, referral, or region predict hire rate?

04 SQL — Advanced Queries

All 10 queries are available on GitHub. Here are two examples showcasing advanced techniques:

Recruitment Funnel — Conversion Rates by Industry SQL 
SELECT  j.industry,
        count(*) as total_candidate,
        sum(case when a.status = 'hired' then 1 end) as hired,
        sum(case when a.status = 'interview' then 1 end) as interview,
        ROUND(CAST(
        sum(case when a.status = 'interview' or a.status = 'hired' then 1 end)
        as float)/cast(count(*) as float)*100,2) as 'Conversion Rate',
        ROUND(CAST(
        sum(case when a.status = 'hired' then 1 end) as float)/
        cast(sum(case when a.status = 'interview'
             or a.status = 'hired' then 1 end) as float)*100,2)
        as 'Candidate Conversion Rate'
FROM jobs j join applications a on j.job_id=a.job_id
GROUP BY j.industry
ORDER BY 'Conversion Rate' asc
Month-over-Month Hires — LAG Window Function SQL 
WITH monthly as (
  SELECT FORMAT(apply_date, 'yyyy-MM') AS month,
         COUNT(*) AS monthly_hires
  FROM applications
  WHERE [status]='hired'
  GROUP BY FORMAT(apply_date, 'yyyy-MM')
)
SELECT month,
       monthly_hires,
     ( cast(monthly_hires as float)
       - cast(LAG(monthly_hires,1) over(ORDER BY month) as float))
     / cast(NULLIF(LAG(monthly_hires,1) OVER (ORDER BY month),0)
       as float)*100 as pct_change
FROM monthly

Why LAG matters here

LAG() retrieves the value from the previous row without a self-join — enabling month-over-month comparison in a single query. NULLIF prevents division by zero when the previous month had zero hires.

05 Key Findings

The dashboard revealed several actionable insights about the recruitment process:

1 Referral Impact

Candidates with referrals are hired at 31% vs 22% without — a 9-point gap showing referrals are the single strongest hiring predictor.

2 Rejection Paradox

48% of rejected candidates met all experience requirements. Competition, not skills gap, is the main rejection driver.

3 Education Effect

PhD holders hired at 34% vs 18% for high school graduates — consistent linear relationship across all levels.

4 Banking Problem

Banking shows both the slowest hiring process (9.9 days avg) and the lowest hire rate (18.8%) — a clear process efficiency issue.

5 Regional Gap

Center region has 5x more applications than South (1,519 vs 266) — but Jerusalem shows the highest hire rate despite lower volume.

6 Interviewer Variance

Top interviewer passes 73.9% of candidates vs 66.8% for the lowest — a 7-point gap suggesting inconsistent evaluation standards.
